Accessing Settings in VS Code

  1. Press Ctrl+, (Windows/Linux) or Cmd+, (Mac)
  2. Search for "GitHub Copilot"
  3. Configure your preferences

Essential Settings

{
  "github.copilot.enable": {
    "*": true,
    "yaml": false
  },
  "editor.inlineSuggest.enabled": true,
  "github.copilot.editor.enableAutoCompletions": true
}

üìù Custom Instructions

Project-Level Instructions

Location: .github/copilot-instructions.md

Steps:

  1. Create .github folder in your repository root
  2. Create copilot-instructions.md inside
  3. Add your project-specific guidelines

Example:

# Project: E-Commerce Platform

## Tech Stack
- Backend: Node.js with Express
- Frontend: React with TypeScript
- Database: PostgreSQL

## Coding Standards
- Use async/await for asynchronous operations
- All functions must have JSDoc comments
- Follow ESLint rules strictly

File-Specific Instructions

Location: .github/instructions/*.instructions.md

Example for Python:

---
description: Python coding standards
applyTo: '**/*.py'
---

# Python Standards
- Follow PEP 8
- Use type hints for all functions
- Write Google-style docstrings

üí¨ Custom Prompts

Creating Reusable Prompts

Location: .github/prompts/*.prompt.md

Steps:

  1. Create .github/prompts folder
  2. Create files with .prompt.md extension
  3. Write your prompts

Using Prompts:

  1. Open Copilot Chat
  2. Type # to see available prompts
  3. Select a prompt or type #filename

Example Prompt:

# create-api-endpoint.prompt.md

Create a REST API endpoint with:
- Method: [GET/POST/PUT/DELETE]
- Route: /api/[resource]
- Input validation
- Error handling
- Unit tests

üîå MCP Servers (Model Context Protocol)

What are MCP Servers?

MCP servers extend Copilot's capabilities by connecting to external tools and data sources.

Popular MCP Servers

  • Azure MCP: Interact with Azure resources
  • Microsoft Learn: Access Microsoft documentation
  • GitHub MCP: Interact with GitHub repositories
  • Database MCP: Query databases

Installing MCP Servers

Method 1: Command Palette

  1. Press Ctrl+Shift+P
  2. Type "MCP: Add Server"
  3. Follow configuration prompts

Method 2: Manual Configuration

{
  "mcp.servers": {
    "azure": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@azure/mcp@latest", "server", "start"]
    },
    "microsoft-learn": {
      "type": "http",
      "url": "https://mcp-server.microsoft.com/learn"
    }
  }
}

üéØ Complete Setup Example

Setting Up a Python Project

1. Create Project Structure

mkdir -p .github/instructions .github/prompts .vscode

2. Create Main Instructions

# .github/copilot-instructions.md
# My Python Project

A Flask REST API with PostgreSQL database.

## Tech Stack
- Python 3.11
- Flask 3.0
- PostgreSQL 15
- pytest for testing

## Standards
- Follow PEP 8
- Use type hints
- 90% test coverage required

3. Create Python-Specific Instructions

# .github/instructions/python.instructions.md
---
description: Python coding standards
applyTo: '**/*.py'
---

# Python Standards
- Follow PEP 8
- Use Black for formatting
- Write Google-style docstrings

4. Configure Workspace

// .vscode/settings.json
{
  "python.linting.enabled": true,
  "python.formatting.provider": "black",
  "editor.formatOnSave": true,
  "python.testing.pytestEnabled": true
}

5. Commit Configuration

git add .github/ .vscode/
git commit -m "Add Copilot customization"
git push
