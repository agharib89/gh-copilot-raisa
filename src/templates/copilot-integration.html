Installation & Setup

1

Install GitHub Copilot

Visual Studio Code

  1. Open VS Code
  2. Go to Extensions (Ctrl+Shift+X / Cmd+Shift+X)
  3. Search for "GitHub Copilot"
  4. Click "Install" on both:
    • GitHub Copilot - Code completion
    • GitHub Copilot Chat - Interactive chat
  5. Sign in with your GitHub account
  6. Authorize the extensions

üí° You need an active GitHub Copilot subscription

JetBrains IDEs (IntelliJ, PyCharm, etc.)

  1. Open your JetBrains IDE
  2. Go to Settings/Preferences (Ctrl+Alt+S / Cmd+,)
  3. Navigate to Plugins
  4. Search for "GitHub Copilot"
  5. Click "Install"
  6. Restart the IDE
  7. Tools ‚Üí GitHub Copilot ‚Üí Login to GitHub

üí° Compatible with all JetBrains IDEs version 2021.2+

2

Configure Copilot Instructions

Add a .github/copilot-instructions.md file to your project to customize Copilot's behavior:

# GitHub Copilot Instructions

This project follows Python best practices and conventions.

## Project Overview

[Describe your project here]

## Coding Standards

- Follow PEP 8 style guidelines
- Use type hints for all function parameters and return values
- Write PEP 257 compliant docstrings
- Include comprehensive error handling
- Write unit tests for all new functionality

## Python Conventions

- Use 4 spaces for indentation
- Keep lines under 79 characters
- Use descriptive variable and function names
- Prefer list comprehensions for simple transformations
- Use context managers for resource management

## Testing Requirements

- Use pytest for all tests
- Follow Arrange-Act-Assert pattern
- Mock external dependencies
- Aim for 80%+ code coverage
- Place tests in tests/ directory with test_ prefix

## Security Practices

- Never hardcode secrets or credentials
- Validate and sanitize all user inputs
- Use parameterized queries for database operations
- Follow principle of least privilege
- Log security-relevant events

## Documentation Standards

- Module-level docstrings explaining purpose
- Function docstrings with Args, Returns, and Raises sections
- Inline comments for complex logic
- Keep README.md up to date
